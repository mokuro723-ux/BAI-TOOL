# BAI Tool — 実装仕様書

## 概要

ブルーアーカイブの総力戦タイムライン（TL）をスマホで素早く視覚化・まとめるためのWebツール。
GitHub Pages（個人アカウント）にデプロイする単一HTMLファイル構成。

---

## 技術スタック

- **単一ファイル構成**: `index.html`（HTML + CSS + JavaScript をすべて inline）
- **外部依存なし**（CDNも不要）
- **PNG出力**: `html2canvas` または Canvas APIで自前実装
- **対象ブラウザ**: iPhone Safari（iOS 15以降）

---

## 機能要件

### 1. タイムライン設定

- ユーザーが上限秒数を自由に入力して設定（例: 90、120、180、240など）
- 横軸が秒数のタイムラインを表示
- 秒単位の目盛りを表示（10秒ごとに主目盛り、適宜間引き）

### 2. キャラクター管理

- 最大10スロット（同キャラ重複あり）
- 各キャラに以下を設定：
  - **アイコン画像**（任意アップロード）または **テキスト＋カラー**（画像なし時の代替）
  - **キャラ名**（ラベル表示用）
  - **表示色**（線・矢印の色に使用）

### 3. スキルアクション配置

- タイムライン上に「スキル使用イベント」を配置
- 各イベントに設定する情報：
  - 使用キャラ（登録済みキャラから選択）
  - 発動秒数
  - スキル対象キャラ（登録済みキャラから選択、複数可）
  - メモ（任意テキスト）
- スキル対象は **矢印／線** で使用キャラアイコン → 対象キャラアイコン へ繋いで表示

### 4. メモ欄

- TL全体に対するメモを自由記述できるテキストエリア
- PNG出力時にメモ欄も含める

### 5. PNG出力

- 「画像として保存」ボタン1つでタイムライン全体をPNG出力
- iPhoneのSafariでダウンロードまたは「写真に保存」できる形式で出力
- 出力サイズはタイムラインが見やすい幅に自動調整（横幅固定、縦は内容に応じて可変）

## UI構成（画面レイアウト）

> **設計変更（実装版）**: 当初は「キャラ別の横軸タイムライン」を想定したが、
> 片手・速記での入力速度を最優先するため、**左列＝値（ゲーム内タイム or コスト）の縦リスト型**に変更した。
> 1 行 = 1 スキルアクション。画面下部の固定バー（親指ゾーン）にキャラアイコンを並べ、
> **タップ 1 回で 1 イベント記録**できる。

```
┌─────────────────────────────┐
│ BAI Tool        [キャラ][設定]│  ← ヘッダー
├─────────────────────────────┤
│ 左列:タイム  TL名:…  並替 消去 │  ← コントロールバー
├─────────────────────────────┤
│ ▲▼ │ 3:30 │ (ホシノ)→(敵)    │  ← イベント行（左列=値）
│ ▲▼ │ 3:25 │ (ヒナ)  EXメモ    │    タップで編集シート
│ ...                          │
├─────────────────────────────┤
│ メモ: [                    ] │  ← メモ欄
├─────────────────────────────┤
│    [📷 画像として保存]        │  ← 出力ボタン
├─────────────────────────────┤
│ [− 4:00 +]  ヒント            │  ← 固定クイックバー
│ (キャラ)(キャラ)(キャラ)…[＋] │    タップ=即記録 / 長押し=詳細
└─────────────────────────────┘
```

### 左列（確定仕様）
- 各イベント行の左端に **ゲーム内タイム（M:SS）** または **コスト** を表示
- どちらを使うかは設定で切替（`mode: time | cost`）
- タイムは下限0でカウントダウン、コストは加算が基本（クイック記録時に値を自動増減）

---

## 入力フロー（速記の核）

1. 事前に「＋キャラ」でキャラを登録（最大10、色/文字/画像）
2. クイックバーで値（タイム/コスト）を合わせる（ステッパー or 入力）
3. **キャラアイコンをタップ → その値で 1 イベント即記録（1 タップ）**
4. 対象キャラ・メモが必要な行だけ、行タップ or アイコン長押しで編集シートを開く

→ 最小 **1 タップ / 1 イベント**。記録後は次の入力に向けて値を自動で1段進める。

## UX要件（iPhone最適化）

- タップ操作でスキルイベントを追加・編集・削除
- 主要操作（クイックバー）は画面下部=親指ゾーンに固定配置
- 全体は縦スクロール、クイックバーのキャラ行は横スクロール
- フォントサイズ・タップターゲットはiOS HIG準拠（最小44px）
- イベント/キャラ編集はボトムシートUI

---

## データ管理

- `localStorage` にTLデータを自動保存（ページリロード後も復元）
- データ形式はJSON（内部管理のみ、エクスポート機能は不要）

---

## ファイル構成

```
index.html   ← すべてを含む単一ファイル
```

GitHub Pagesにそのままプッシュすれば動作する構成にすること。
