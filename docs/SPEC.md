# BAI Tool — 実装仕様書（現行・正典）

> このファイルが実装の正典。仕様変更時はコードと本ファイルの両方を更新する。
> 過去の経緯は書かない。**現状目指しているもの**だけを記述する。

## 目的・前提

- ブルーアーカイブ総力戦の TL（タイムライン）を、**スマホで片手で素早く「速記」**し、
  あとから見て分かりやすい**表**として **PNG 画像出力**するツール。
- 既存の誰かの TL を記録する用途（TL を考案するツールではない）。
- 想定環境: **iPhone Safari（iOS 15+）/ スマホ幅 375px**。作者個人が GitHub Pages で公開。

## 最優先の設計原則（他に優先する）

1. **入力速度 > 機能の多さ** — 1 配置のタップ数を最小に。文字入力を極力避ける。
2. **片手・スマホ前提** — 主要操作は画面下部（親指ゾーン）。タップ最小 44px。

## 技術・構成

- **単一ファイル `index.html`**（HTML+CSS+JS を inline）。**外部ライブラリ・CDN なし**。バニラ JS。
- 保存: `localStorage`（キー `bai-tool-store-v1`）に JSON 自動保存。エクスポート機能は不要。
- PNG 出力は Canvas API で自前実装。

## データモデル

```
store = { docs:[doc], currentId, imgLib:{ 名前: dataURL } }   // imgLib は全TL共通
doc   = { id, title, roster:[char], rows:[row], activeId, updatedAt }
char  = { id, name, text, color, img(dataURL|null), num }     // roster = 今回使うキャラ(最大10)
row   = { id, num, time, cells:[cell], memo }                 // 表の1行
cell  = { charId, badge:(number|null), ring:(色|null) }       // 配置した1キャラ
```

- `state` = 現在表示中の `doc`。`save()` は `store` 全体を保存。
- 旧 `bai-tool-v1`（単一データ）は起動時に自動移行する。

## 機能要件

### 1. 複数 TL（ドキュメント）
- `☰ TL` で一覧。各 TL を **開く / 保存(=画像出力) / 削除**、**＋新規 TL** 作成。
- 一覧に更新日時・行数を表示。最後の 1 件は削除不可。

### 2. 使用キャラ（roster, 最大10）
- ヘッダ `キャラ` から管理。各キャラ: **名前 / アイコン文字(最大2) / 表示色 / 画像**。
- **名前検索**: 内蔵リスト `CHAR_NAMES`（ひらがな可・部分一致）から選択で名前と2文字を自動設定。
- **画像**: ①手動アップロード ②**SchaleDB 公式アイコン自動取得**（名前選択時に自動 or ボタン）
  ③**保存済みアイコンの再利用**（imgLib、名前キー）。取得画像は端末ローカルのみ保存。

### 3. 表の編集
- 行: コスト / タイム / キャラ配置 / メモ。`＋行`で追加、`▲▼`で移動、`行複製`/`行削除`。
- **選択中(active)の行**が青ハイライト。各セルや入力にフォーカスで active 切替。
- **コスト欄**: 整数のみ丸数字表示（7→⑦）。小数・「即」等はそのまま（フォーカス中は生値表示）。
- **タイム欄・メモ欄**: 自由入力（手動）。
- **空行**は段落区切りとしてそのまま使える。

### 4. キャラ配置（速記の核）
- 下部ロスターのキャラを**タップ → 選択中の行に配置**（1 タップ）。
- 置いたセルのジェスチャ:
  - **タップ = 枠色 cycle**（スキル対象の判別用。既定=黒 → 白→赤→黄→緑→水色→ピンク→橙）
  - **上 = badge +1 / 下 = -1 / 右 = +3 / 左 = 削除**
- アイコンは**小さめ・角丸四角・既定の外枠は黒**。判別がつけば良い。

### 5. PNG 出力
- 「画像として保存」または TL 一覧の「保存」で、表全体を PNG 化。
- iOS: 生成画像をシート内に表示 →**長押しで写真に保存**（PC はダウンロードボタン）。
  ※ iOS は非同期の `window.open`/ダウンロードを弾くため、この方式は必須。

### 6. TL取込（テキスト貼り付け / 画像OCR）
- ヘッダーの 📋 ボタンから取込シートを開く。
- **テキスト貼り付け**: Discord/X等からコピペしたTLテキストをそのままパース（OCR不要・高精度）。
- **画像から読込**: スクショしかない場合、`Tesseract.js`（CDN動的読込、jpn+eng）でOCRし、
  認識結果をテキスト欄に展開。**OCRは誤読する前提**で、必ず人がテキスト欄を見て確認・修正してから
  「解析→反映」を押す運用（自動で直接TLに書き込まない）。
- パース仕様（`parseTLImportText`）:
  - 1行の `→` はステップ区切りとして複数 row に分割。
  - 空行はそのまま空 row（段落区切り）。
  - 行頭の `AUTO/即/後掛/※/NS/ギブアップ/数字` 等の記号を `marker` として抽出し、
    数字は `row.num`、記号テキストは暫定で `row.memo` に格納
    （**将来的に専用表示欄を作る予定。現状はメモ欄が暫定置き場**）。
  - 残りのテキストを `,`/`、` で分割しキャラ名候補とし、`state.roster` に対し
    完全一致→前方一致で照合して `cells` に配置。一致しない名前は `row.memo` に
    テキストとして残し未配置（ロスター未登録キャラは事前に手動登録しておく運用前提）。
  - **既知の限界**: 「5 チャージ①」のように、直前の行のキャラ名を省略して
    スキル名だけが書かれる行は、現状は前の行のキャラに自動で結びつけず
    `memo` に残すだけ（要手動修正）。
- 反映は既存行を上書きせず、**TLの末尾に追加**（破壊的操作を避ける）。

## 自動更新（キャラ名・アイコン）

- `index.html` の `CHAR_NAMES`（名前リスト）と `CHAR_ICONS`（名前→SchaleDB Id）は、
  `/* CHAR_NAMES:START */`〜`END`、`/* CHAR_ICONS:START */`〜`END` で囲む。**マーカーを壊さない**。
- `scripts/update-chars.mjs` が **SchaleDB**（`data/jp/students.min.json`）から取得し両方を置換。
  GitHub Actions が毎月1日に自動実行・自動コミット。手動は `node scripts/update-chars.mjs`。
- アイコン取得元: `https://schaledb.com/images/student/icon/{Id}.webp`（CORS `*` のため PNG 出力可）。
