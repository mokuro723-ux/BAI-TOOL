## このセッションで行った変更（最新、すべて index.html のみ）

ユーザー報告: 「チャージ①」のようなキャラ名省略行で、チャージ先のアイコンが
直前のキャラ（カヨコ）のままになり、本来のチャージ対象（アリス）にならない。
目指す出力は IMG_2986.png のような見た目（行は圧縮されてOK、チャージ先だけ正しければよい）。

### A. スキルキーワード辞書を新設（バグの根本原因への対応）

- 前セッションまでの実装は「キャラ名が省略された行＝直前に配置したキャラ
  (`lastChar`)のスキル」と機械的に決め打ちしていたため、「カヨコ→チャージ①→チャージ②」
  のような並びで、チャージ先が常に直前のカヨコになってしまっていた。
- ユーザーへのヒアリングの結果、「チャージ」のような効果名キーワードと紐付け先キャラは
  本来 `lastChar` 自動推測ではなく、**辞書で明示的に設定するもの**という方針が確定。
  （今はチャージ持ちはアリスのみだが、今後増えた場合に備えて拡張しやすい形にした）
- `bai-tool-skill-keywords`（localStorage）に `{"チャージ":"アリス"}` の形で保存。
- `resolveSkillKeyword(nm)`: nm にキーワードが**含まれていれば**（前方一致ではなく部分一致）
  対応するロスターキャラを返す。長いキーワード優先。
- キャラ名省略行の解決順序を `スキルキーワード辞書 → lastChar` に変更
  （辞書にヒットすればそちらを優先、なければ従来通り直前キャラにフォールバック）。
  「スキル名のみ行を直前行に統合する」処理、および同一行内の省略セグメント処理の
  両方に同じ優先順位を適用。
- 📖ボタンの画面（旧:あだ名辞書シート）に「スキルキーワード」セクションを追加。
  キーワード・対象キャラのプルダウンを選んで「追加」、一覧から個別「削除」が可能。
  ヘッダーのボタンは増やさず既存の📖に統合（ボタン数を抑えるため）。

### B. 行の途中にマーカー単語が来るケースの分割漏れを修正

- ユーザー提供のサンプルで新たに発覚したバグ。
  `通常攻撃1発後 即 カヨコ 即キサキ` のように、行の途中（先頭以外）に
  `即/AUTO即/後掛` 等のマーカー単語が複数回出てくる入力で、
  `splitMarker` が行頭からの連続マーカーしか見ておらず、`即 カヨコ 即キサキ` の
  部分が丸ごと1つの未マッチ文字列として扱われ、**キャラが1人も配置されないまま**
  直前行に吸収されてしまっていた。
- `splitBodyIntoSegments(body)` を新設。MARK_TOKENS の出現位置（行頭以外も）で
  body をサブセグメントに分割し、各塊の先頭マーカーを剥がした上で
  スペース/カンマ/読点区切りで複数キャラ名候補を取り出す。
  ロスター名にもスキルキーワードにも一致しない断片は自由文としてメモに集約。
- 呼び出し側では、body の途中にマーカー単語が含まれる場合のみこの新ロジックを使い、
  含まれない通常パターン（`アリス,キサキ,アコ〆` 等）は従来の単純な
  カンマ/読点分割のまま（既存動作への影響を避けるため）。

### 確認方法・テスト結果

- ユーザー提供の実サンプル（AUTO即/カヨコ/チャージ①/チャージ②/…/ギブアップ、14行）で
  Playwright実ブラウザ実行・取込シートUI経由での反映・スクリーンショット確認済み。
  チャージ①②のアイコンが正しくアリスになることを確認。
- 行の圧縮（複数入力行→1出力行）自体はユーザー許容範囲内（「右下に数字出せるんだから
  圧縮していい、見やすければいい」と回答済み）。ただし「後掛即」の次の行に
  4入力分（カヨコ・キサキ・チャージ①・チャージ②）が1行にまとまる点は、
  ユーザー未確認（「後で聞け」と保留）。**次回セッションで確認すること**。
- 既存機能（行追加連打のスクロール対策／Undo・Redo／PinP余白トグル）の回帰テストも実施、
  異常なし。
- 自動更新ロジック（`CHAR_NAMES`/`CHAR_ICONS`/`iconUrlFor`/`autoIcon`/`imageUrlToDataUrl`）
  には一切触れていない。構文チェック（`node -e "new Function(...)"`）OK。

## 過去セッションでの変更（参考）

### 1. findRosterChar の強化（部分一致・あだ名解決）

照合順序を以下に変更：
1. あだ名辞書で正式名に変換 → 完全一致
2. 完全一致
3. 入力名がロスター名で始まる前方一致（最長一致優先）
4. 括弧前のベース名が一致するキャラがロスターに1件だけ → 確定マッチ
   例: ロスターに「アリス(臨戦)」のみ → 「アリス」「アリスチャージ」で確定一致

### 2. あだ名辞書（localStorage: "bai-tool-nicknames"）

- `loadNicknames()` / `saveNicknames(d)` / `resolveNickname(name)` を追加
- 構造: `{ "クロコ": "シロコ(テラー)", "アリス": "アリス(臨戦)" }`
- TLデータ（bai-tool-store-v1）とは別キーで独立保存

### 3. TL取込シートの未マッチ紐付けUI

- 「解析→反映」後、未マッチ名が存在する場合に紐付けパネルを表示
- ドロップダウンでロスターキャラを選択 → 「辞書に保存して再解析」で即反映

### 4. ヘッダーにあだ名管理ボタン（📖）追加

- `btnNick`（📖）を追加、`openNickSheet()` を開く
- 辞書の一覧・個別削除・全削除が可能

### 5. splitMarker の修正（丸数字をコストとして認識）

**修正前**: `⑤ チャージ①` → `⑤` がキャラ名扱いになりマッチ失敗
**修正後**: 丸数字（①-⑳・⓪）を行頭コスト数字として最優先で取り込む

### 6. zenkakuToNum の修正（丸数字→半角数字変換）

**修正前**: 丸数字をコスト欄の数字に変換しておらず、コスト欄が空になっていた
**修正後**: 丸数字を半角数字に変換してから `row.num` に格納

### 7. スキル名のみ行の処理（別行を作らず前の行に追加）

**問題**: `⑤ チャージ①` → marker=`⑤`、rest=`チャージ①` と正しく分解されるが、
`チャージ①` はキャラ名でないため「別の新規行」に押し出されていた。

**修正後**: rest がキャラ名でなく `lastChar` が存在する場合、
新規行を作らず**直前の行（rows末尾）にセルを追加**する。
コスト（⑤）も直前行に上書きする。

### 8. ※ の扱い修正

**修正前**: `※` を `NOTE_PREFIXES` に含めていたため、
`※即 チャージ①` が noteLine 判定されメモ欄に入っていた。

**修正後**: `※` を `NOTE_PREFIXES` から除外。
`※` は `MARK_TOKENS` で marker として処理し、後ろにスキル名が続く場合は通常行として扱う。
`※` 単独行（rest が空）はメモ欄に入る（正常動作）。

### 9. AUTO即・即・後掛即等のマーカーをメモ欄に表示

**修正前**: `AUTO即` 単独行はコスト欄もメモ欄も空になっていた（どこにも表示されなかった）
**修正後**: marker のトークン部分（AUTO即・即・後掛即・NS等）をメモ欄に表示する

### 10. セルの左上ラベル（mark）の整理

- `splitNameAndBadges` が返す `mark` をそのまま使うよう統一（呼び出し側で再計算しない）
- suffix（「チャージ」等）: 最大3文字で左上ラベルに表示
- Cプレフィックス（`C`・`Ｃ`・`©`・`©︎`）: 「C」として左上ラベルに表示
- 組み合わせ例: `Cアリスチャージ` → 左上「Cチャ」

## パース処理の全体フロー（最新）

```
入力テキスト1行
  ↓ isNoteLine() で注意書き判定（※はここでは判定しない）
  ↓ "→" で分割（複数セグメント対応）
  ↓ splitMarker() でコスト/マーカーと残りに分解
      - 丸数字①② → コスト数字として取り込み
      - 半角/全角数字 → コスト
      - MARK_TOKENS（AUTO即・即・後掛即・※・NS等）→ markerLabel
  ↓ zenkakuToNum() で丸数字・全角数字 → row.num
  ↓ markerLabel → メモ欄に追加
  ↓ rest がスキル名のみ（キャラ名でない）かつ lastChar がいる
      → 新規行を作らず直前行にセル追加、コストも直前行に上書き
  ↓ rest をカンマ/読点で分割 → names[]
  ↓ 各 name に splitNameAndBadges() を適用
      - C/©プレフィックス → mark="C"
      - findRosterChar() でキャラ特定
      - 丸数字 → badge
      - 残り文字列 → mark（最大3文字）
  ↓ マッチしない name かつ lastChar がいる → lastChar のセルとして追加
  ↓ マッチしない name かつ lastChar なし → unmatched[]
```
