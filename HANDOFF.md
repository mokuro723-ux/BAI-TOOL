## このセッションで行った変更（最新、すべて index.html のみ）

ユーザー報告: 何度スキルキーワード辞書で「チャージ→アリス」と設定しても、
チャージ先のアイコンがアリスにならない。実機スクショ（IMG_3048.png）で確認。

### 根本原因（2つ複合していた）

1. **「スキル名省略行」とみなす条件が広すぎた**
   - 丸数字（①②）もスキルキーワードも含まない、ただの自由文コメント
     （例:「通常攻撃1発後」）まで「キャラ名省略＝直前キャラの続き」と誤判定し、
     直前の行（カヨコ/キサキの行）に無条件で吸収していた。
   - その結果、本来アリスのチャージとして扱われるべき行が、先に自由文コメントに
     よって直前行の構造が壊された後段で処理され、見た目上カヨコ/キサキの行に
     紐付いてしまっていた（チャージ自体の紐付けロジックは正しく動いていたが、
     その手前で行が壊れていたため結果的に「効かない」ように見えていた）。
   - **修正**: 「スキル名省略行」とみなす条件を「丸数字を含む」または
     「スキルキーワード辞書に一致する」場合のみに限定。それ以外の自由文は
     直前キャラに吸収せず、独立した行（メモのみ）として残すようにした。
     （`parseTLImportText` 内、早期判定ブロックと行内ループの両方を修正）

2. **スキルキーワード辞書が空のまま運用されていた**
   - 辞書はlocalStorageに保存する設計のため、ユーザーが📖画面で一度も
     保存していない・別環境で試した等の場合は空辞書のままで、
     「チャージ→アリス」のルール自体が存在せず効きようがなかった。
   - **修正**: 初回（未設定時）に既定値 `{"チャージ":"アリス"}` を自動投入する
     ようにした（`loadSkillKeywords()`）。📖画面から変更・追加・削除は従来通り可能。

3. **（付随バグ）直前行への吸収時、その行のmarker/自由文（NS・発射確認等）が
   そのまま消えていた**
   - 吸収時に `continue` するだけでメモへの追記をしておらず、
     「NS 発射確認」のような情報がどこにも残らず消失していた。
   - **修正**: 吸収する際も marker・自由文を対象行の `memo` に追記するようにした。

### テスト結果

ユーザー提供の実テキスト（AUTO即/カヨコ/チャージ①/チャージ②/…/ギブアップ、14行）を
Node上で `parseTLImportText` 単体テストして確認：
- 「カヨコ」行の後の「チャージ①」「チャージ②」は **正しくアリスに**（badge 1,2、左上ラベル「チャ」）。
- 「即 チャージ①」「NS 発射確認 即 チャージ②」も同様に正しくアリスへ紐付き。
- 「通常攻撃1発後」のような自由文コメントは直前キャラに吸収されず、
  独立行 or メモへの追記のみになり、行の構造を壊さない。
- 構文チェック（`node -e "new Function(...)"`）OK。

### 既知の細かい点（実害は小さいが記録）

- 「通常攻撃1発後」のような自由文コメントは、ロスター名と一致しないため
  TL取込シートの「未マッチ警告」欄にも出る（実際はキャラ名ではなく注釈なので
  ユーザーは無視してよいが、表示上は「未配置キャラ」として警告されてしまう）。
  気になる場合は次回、注釈っぽい自由文を未マッチ警告から除外する改善を検討。
- ブラウザ実機（iPhone Safari）で📖画面の既定値投入・実際のOCR取込導線は未検証
  （Node単体テストのみ）。次回触る際は実機確認を。

## 過去セッションでの変更（参考）

### 1. findRosterChar の強化（部分一致・あだ名解決）

照合順序を以下に変更：
1. あだ名辞書で正式名に変換 → 完全一致
2. 完全一致
3. 入力名がロスター名で始まる前方一致（最長一致優先）
4. 括弧前のベース名が一致するキャラがロスターに1件だけ → 確定マッチ
   例: ロスターに「アリス(臨戦)」のみ → 「アリス」「アリスチャージ」で確定一致

### 2. あだ名辞書（localStorage: "bai-tool-nicknames"）

- `loadNicknames()` / `saveNicknames(d)` / `resolveNickname(name)` を追加
- 構造: `{ "クロコ": "シロコ(テラー)", "アリス": "アリス(臨戦)" }`
- TLデータ（bai-tool-store-v1）とは別キーで独立保存

### 3. TL取込シートの未マッチ紐付けUI

- 「解析→反映」後、未マッチ名が存在する場合に紐付けパネルを表示
- ドロップダウンでロスターキャラを選択 → 「辞書に保存して再解析」で即反映

### 4. ヘッダーにあだ名管理ボタン（📖）追加

- `btnNick`（📖）を追加、`openNickSheet()` を開く
- 辞書の一覧・個別削除・全削除が可能

### 5. splitMarker の修正（丸数字をコストとして認識）

**修正前**: `⑤ チャージ①` → `⑤` がキャラ名扱いになりマッチ失敗
**修正後**: 丸数字（①-⑳・⓪）を行頭コスト数字として最優先で取り込む

### 6. zenkakuToNum の修正（丸数字→半角数字変換）

**修正前**: 丸数字をコスト欄の数字に変換しておらず、コスト欄が空になっていた
**修正後**: 丸数字を半角数字に変換してから `row.num` に格納

### 7. スキル名のみ行の処理（別行を作らず前の行に追加）

**問題**: `⑤ チャージ①` → marker=`⑤`、rest=`チャージ①` と正しく分解されるが、
`チャージ①` はキャラ名でないため「別の新規行」に押し出されていた。

**修正後**: rest がキャラ名でなく `lastChar` が存在する場合、
新規行を作らず**直前の行（rows末尾）にセルを追加**する。
コスト（⑤）も直前行に上書きする。

### 8. ※ の扱い修正

**修正前**: `※` を `NOTE_PREFIXES` に含めていたため、
`※即 チャージ①` が noteLine 判定されメモ欄に入っていた。

**修正後**: `※` を `NOTE_PREFIXES` から除外。
`※` は `MARK_TOKENS` で marker として処理し、後ろにスキル名が続く場合は通常行として扱う。
`※` 単独行（rest が空）はメモ欄に入る（正常動作）。

### 9. AUTO即・即・後掛即等のマーカーをメモ欄に表示

**修正前**: `AUTO即` 単独行はコスト欄もメモ欄も空になっていた（どこにも表示されなかった）
**修正後**: marker のトークン部分（AUTO即・即・後掛即・NS等）をメモ欄に表示する

### 10. セルの左上ラベル（mark）の整理

- `splitNameAndBadges` が返す `mark` をそのまま使うよう統一（呼び出し側で再計算しない）
- suffix（「チャージ」等）: 最大3文字で左上ラベルに表示
- Cプレフィックス（`C`・`Ｃ`・`©`・`©︎`）: 「C」として左上ラベルに表示
- 組み合わせ例: `Cアリスチャージ` → 左上「Cチャ」

## パース処理の全体フロー（最新）

```
入力テキスト1行
  ↓ isNoteLine() で注意書き判定（※はここでは判定しない）
  ↓ "→" で分割（複数セグメント対応）
  ↓ splitMarker() でコスト/マーカーと残りに分解
      - 丸数字①② → コスト数字として取り込み
      - 半角/全角数字 → コスト
      - MARK_TOKENS（AUTO即・即・後掛即・※・NS等）→ markerLabel
  ↓ zenkakuToNum() で丸数字・全角数字 → row.num
  ↓ markerLabel → メモ欄に追加
  ↓ rest がスキル名のみ（キャラ名でない）かつ lastChar がいる
      → 新規行を作らず直前行にセル追加、コストも直前行に上書き
  ↓ rest をカンマ/読点で分割 → names[]
  ↓ 各 name に splitNameAndBadges() を適用
      - C/©プレフィックス → mark="C"
      - findRosterChar() でキャラ特定
      - 丸数字 → badge
      - 残り文字列 → mark（最大3文字）
  ↓ マッチしない name かつ lastChar がいる → lastChar のセルとして追加
  ↓ マッチしない name かつ lastChar なし → unmatched[]
```
