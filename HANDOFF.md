### 追加: 元に戻す/やり直し（Undo/Redo）（前回セッション）

- ヘッダー右上に `↶`(元に戻す) / `↷`(やり直し) ボタンを追加。
- 仕組み: `snapshotHistory()` を行・セル操作系のボタン onclick の先頭、および
  文字入力欄（コスト/タイム/メモ/タイトル）の `focus` 時に呼び、
  `{title, roster, rows, activeId}` の JSON スナップショットを `undoStack` に積む。
  `undo()`/`redo()` で `undoStack`/`redoStack` 間をやり取りし `applySnapshot()` で復元。
  履歴上限は50件（`HISTORY_LIMIT`）。
- 対象は表示中の TL（`state`＝doc）単位。`switchDoc()` で TL を切り替えると
  `resetHistory()` で履歴をクリアする（別 TL の履歴を持ち越さない）。
- 文字入力は1文字ごとに記録すると使い物にならないため、`oninput` ではなく
  `focus`（編集開始前のタイミング）で1回だけ記録する設計にしている。
- `render()` 内で `updateUndoRedoButtons()` を呼び、履歴が空の側のボタンを
  `disabled` にしている。

### 追加: PinP用余白トグル

- ヘッダーに `⬓` ボタンを追加。タップで `#pinpSpacer`（ヘッダー直下の空div）の
  高さを 0 ⇔ 300px に切り替える。
- 背景: ユーザーがスマホの PinP（ピクチャインピクチャ）で参考動画を画面上部に
  重ねながら TL を記録する運用のため、PinP表示中だけ表の先頭行を動画の下に
  隠れないよう下げたい、という要望。常時固定余白ではなくワンタップ切替式を採用。
- 状態は `localStorage`（キー `bai-tool-pinp-gap`）に保存。TLデータ
  （`bai-tool-store-v1`）とは別キーで、端末の表示設定として独立させている。
  リロードしても直前の表示状態を復元する。
- `docs/SPEC.md` の画面レイアウト図・機能要件にも追記済み。

### 修正: 行追加連打でスクロール不能になるバグ（前回セッション）

- **症状**: 「＋行」ボタンを連続でタップすると、画面が自動スクロールを繰り返して
  操作を受け付けなくなる（入力不可能になる）。
- **原因**:
  `index.html` の `btnAddRow` の `onclick` 内で、行追加のたびに
  `renderTable()`（テーブル全体を作り直す重い処理）→
  `scrollIntoView({behavior:"smooth"})`（アニメーション付きスクロール）
  を実行していた。連打すると、前のスムーズスクロールのアニメーションが終わる前に
  次のスクロール要求が割り込み、スクロール位置の競合が起きてブラウザ側の
  スクロール処理が暴走 → 指のタップ座標とスクロール位置がズレ続けてタップが
  効かなくなる、という流れ。
- **修正内容**:
  - `scrollIntoView` の `behavior` を `"smooth"` → `"instant"`（瞬間移動）に変更。
  - 加えて `scrollRowIntoView()` という関数を新設し、`requestAnimationFrame` で
    スクロール実行を1フレームに1回だけにまとめるようにした（連打時の多重実行防止）。
  - 変更箇所は `index.html` 内、`/* ===== row ops ===== */` コメント直後、
    `btnAddRow` の `onclick` 周辺のみ。他の機能・他のボタンには触れていない。
- **確認方法**: 「＋行」を連打しても画面が暴れずに最終行までジャンプし、
  直後にタップ・入力操作が即座に効くこと。スマホ幅375pxで確認すること。

## 変更してはいけないもの（再掲・最重要）

[CLAUDE.md](CLAUDE.md) にある通り、以下は **本ツールの中核機能** であり、
仕様変更・リファクタの際も弱める/無効化する変更は禁止。変更が必要な場合は
実装前に必ずユーザーに確認を取ること。

1. **キャラ名/アイコンの月1自動更新**
   （GitHub Actions + `scripts/update-chars.mjs`、SchaleDB `data/jp/students.min.json` から取得）
2. **名前選択時のアイコン自動取得**
   （`iconUrlFor` / `autoIcon` / `imageUrlToDataUrl`。取得先は SchaleDB、保存先は端末の
   localStorage のみ。リポジトリに同梱・再配布はしない）

`CHAR_NAMES:START`〜`END`、`CHAR_ICONS:START`〜`END` のマーカーも壊さないこと。

## このプロジェクト共通のコーディング方針

- 単一 `index.html` のみで完結。新規ファイルに分割しない。
- 外部ライブラリ・CDN を追加しない（バニラJS）。
- 対象環境: iPhone Safari（iOS 15+）、スマホ幅375px。**必ずこの幅で見た目を確認する**。
- コメントは日本語。
- 確認コマンド（構文チェックのみ。実行はしない）:
  ```
  node -e "new Function(require('fs').readFileSync('index.html','utf8').match(/<script>([\s\S]*)<\/script>/)[1])"
  ```
- 仕様変更をしたら `docs/SPEC.md` も合わせて更新し、正典を保つこと。
- 既存機能を壊した場合・未指示の変更を加えた場合、ユーザーは強く不満を示す。
  **指示された範囲だけを過不足なく直すこと。**
